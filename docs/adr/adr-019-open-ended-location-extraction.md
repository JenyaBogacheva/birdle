# ADR-019: Open-Ended Location Extraction Without Closed Lists

- **Status**: Accepted
- **Supersedes**: Partially amends ADR-015 (LLM-Based Location Extraction)
- **Context**: The initial LLM-based location extraction prompt (ADR-015) included a closed list of example country codes (AU, GB, CA, NZ, IN, BR, FR, DE, ES, IT, JP, CN, MX, ZA, AR, CL, PE, CO, EC, VE, KE, TZ, UG, ZW, BW, MY, TH, PH, ID). This created a pseudo-hardcoded mapping that contradicted the original goal of using LLM extraction to avoid maintenance overhead. Users in countries not listed (e.g., Serbia, Poland, Czech Republic) experienced failures when entering well-known city names like "Belgrade" without country qualifiers. The closed list approach required code updates whenever coverage gaps were discovered.
- **Decision**: Remove the closed country list and trust the LLM's comprehensive geographic knowledge. The prompt now references the ISO 3166-1 alpha-2 standard directly without enumerating countries. Added explicit rule: "For well-known cities without country: use the most populous/famous location" to handle ambiguous cases (e.g., "Belgrade" → RS). Reserve "UNKNOWN" response only for truly vague inputs like "downtown", "the park", or "somewhere". Include diverse geographic examples (Belgrade → RS, Warsaw → PL) to demonstrate worldwide coverage.
- **Consequences**: The system now handles any location worldwide without code maintenance. Users can enter city names from any country, and the LLM will intelligently map them using its built-in geographic knowledge. This fully realizes the original ADR-015 vision of flexible, maintainable location handling. Risk: The LLM might occasionally misidentify ambiguous locations, but the added rule to prefer well-known locations mitigates this. The open-ended approach scales to all current and future countries without developer intervention. Cost remains one OpenAI API call per request (~$0.00002). Validates that ADR-015's architectural choice to use LLM extraction was correct, and this change removes an artificial constraint.
