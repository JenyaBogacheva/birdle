# Prompt 5. Workflow Rules — @workflow.md

Create `docs/workflow.md` with simple rules for working through `@tasklist.md`, based on `@vision.md`.

Workflow rules:

- follow the plan strictly
- before each iteration: propose the solution briefly (with small code snippets if needed) and wait for approval
- after approval: implement the iteration
- after implementation: wait for confirmation
- update progress in `@tasklist.md` and mark completed tasks
- confirm transition to the next iteration
- make a commit after each completed iteration

After generating `docs/workflow.md`:

- add this prompt (Prompt 5) to the project’s prompt session.
